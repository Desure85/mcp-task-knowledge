# Обзор репозитория и точки расширения

## Обзор архитектуры

Репозиторий представляет собой **MCP (Model Context Protocol) сервер**, предназначенный для управления задачами и базой знаний. Основные характеристики:

- **Хранилище**: Файловое (JSON для задач, Markdown для знаний).
- **Поиск**: Гибридный (лексический BM25 + векторный поиск через ONNX runtime).
- **Интеграции**: Obsidian (импорт/экспорт), Service Catalog (удаленный/встроенный/гибридный).
- **Промпты**: Продвинутая система управления промптами, включая сборку воркфлоу, A/B тестирование и многорукие бандиты.

## Сильные стороны

1. **Модульность**: Четкое разделение на модули (`storage`, `search`, `catalog`, `prompts`).
2. **Гибкость поиска**: Использование гибридного поиска позволяет находить информацию как по ключевым словам, так и по смыслу.
3. **Готовность к CI/CD**: Наличие скриптов для валидации, индексации и сборки артефактов.
4. **Интероперабельность**: Глубокая интеграция с Obsidian позволяет использовать сервер в привычном для многих workflow.

## Точки расширения

### 1. Слой хранения (Storage Layer)

Текущая реализация жестко завязана на файловую систему.

- **Абстракция**: Внедрение интерфейса `IStorage<T>` позволит отвязаться от файлов.
- **Новые бэкенды**: Поддержка SQLite (для локальной работы, но с транзакциями), PostgreSQL или MongoDB для серверных инсталляций.
- **Облачные адаптеры**: Использование S3 или Google Cloud Storage для хранения вложений или крупных документов знаний.

### 2. Поиск и векторизация

Интерфейс `VectorSearchAdapter` уже существует, что упрощает расширение.

- **Cloud Embedding API**: Добавление адаптеров для OpenAI (`text-embedding-3-small`), Anthropic или Cohere.
- **Внешние Vector DB**: Интеграция с Pinecone, Milvus, Qdrant или Weaviate для работы с большими объемами данных.
- **Reranking**: Добавление второго этапа поиска с использованием Cross-Encoders для повышения точности.

### 3. Интеграции (Новые MCP инструменты)

- **Коммуникации**: Инструменты для взаимодействия со Slack, Microsoft Teams или Telegram (уведомления о задачах).
- **Трекеры**: Интеграция с Jira, GitHub/GitLab Issues для синхронизации задач.
- **Документация**: Поддержка Notion или Confluence как альтернативных источников знаний.

### 4. Воркфлоу и промпты

- **Сложные шаги**: Добавление условий (if-else) и циклов в `prompts_build`.
- **Human-in-the-loop**: Возможность приостановки сборки для ручного подтверждения или правки промежуточного Markdown.
- **Remote Registry**: Возможность подтягивать промпты из удаленного Git-репозитория или реестра.

### 5. Ядро и автоматизация

- **Система плагинов**: Формальный механизм динамической загрузки инструментов из внешних JS-модулей.
- **Событийная модель (Hooks)**: Система событий `onTaskCreated`, `onKnowledgeUpdated`, позволяющая вешать триггеры на изменения (например, авто-тегирование через LLM).
- **Versioning**: Полноценное версионирование документов знаний (хранение истории изменений).

### 6. Наблюдаемость (Observability)

- **Telemetry**: Интеграция с OpenTelemetry для трассировки запросов.
- **Prompt Analytics**: Сбор детальной статистики по стоимости, латентности и качеству (score) для всех используемых промптов.
